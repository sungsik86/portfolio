CASE STUDY · DJANGO · FLY.IO

Fly.io로 Django 웹 페이지 배포하기

Django로 만든 웹 페이지를 개발 환경에서 운영 URL까지 올릴 때 필요한 흐름을 단계별로 정리한 실전 배포 기록입니다. 설정 파일, 실행 명령, 점검 포인트, 실패 케이스 대응까지 한 페이지에서 확인할 수 있도록 구성했습니다.

1. 목표와 전제

목표는 Django 템플릿 기반 웹 페이지를 Fly.io에 올리고, 이후 수정 사항을 flyctl deploy 한 번으로 재배포할 수 있는 상태를 만드는 것입니다. 로컬 실행과 운영 실행의 차이를 줄이기 위해 컨테이너 기반으로 배포합니다.

2. 준비물

  • Python 3.11 이상
  • Django 프로젝트(templates/static 구조 분리)
  • Fly.io 계정 + flyctl 설치
  • GitHub 저장소(버전 관리 및 배포 이력 추적)

3. 핵심 파일 구성

manage.py
portfolio/
  templates/portfolio/index.html
  templates/portfolio/flyio_deploy.html
  static/portfolio/styles.css
  static/portfolio/script.js
portfolio_site/settings.py
Dockerfile
start.sh
fly.toml

4. settings.py 필수 설정

운영 배포에서는 환경변수 기반 설정이 중요합니다. 특히 ALLOWED_HOSTS, STATIC_ROOT가 빠지면 배포 후 400/정적파일 오류가 발생할 수 있습니다. 

SECRET_KEY = os.getenv(\"DJANGO_SECRET_KEY\", \"dev-key\")
DEBUG = os.getenv(\"DEBUG\", \"False\").lower() in {\"1\", \"true\", \"yes\", \"on\"}
ALLOWED_HOSTS = os.getenv(\"ALLOWED_HOSTS\", \"*\").split(\",\")
STATIC_URL = \"static/\"
STATIC_ROOT = BASE_DIR / \"static\"

5. 컨테이너 시작 순서

앱 시작 전에 데이터베이스 마이그레이션과 정적파일 수집을 실행하고, 마지막에 gunicorn으로 앱을 띄웁니다. 

python manage.py migrate --noinput
python manage.py collectstatic --noinput
gunicorn portfolio_site.wsgi:application --bind 0.0.0.0:8000 --workers 2

6. fly.toml에서 맞춰야 할 값

  • internal_port = 8000 (앱 실행 포트와 동일하게)
  • ALLOWED_HOSTS에 *.fly.dev 도메인 반영
  • 필요 시 statics로 정적 경로 매핑

7. 실제 배포 명령

flyctl auth login
flyctl deploy
flyctl status
flyctl logs

배포가 성공하면 https://<app-name>.fly.dev 주소로 페이지가 노출됩니다.

8. 자주 겪는 문제와 해결

1) Repository not found

원격 주소가 잘못 연결된 경우입니다. git remote -v 확인 후 git remote set-url origin ...으로 수정합니다.

2) 배포 후 정적 파일 미반영

collectstatic 누락 또는 경로 설정 불일치가 원인입니다. STATIC_ROOT와 start 스크립트 실행 순서를 점검합니다.

3) Expected address 경고

앱 바인딩 주소/포트와 Fly 내부 포트가 맞지 않으면 발생합니다. 0.0.0.0:8000 + internal_port=8000 조합으로 통일합니다.

9. 검증 체크리스트

  • flyctl status에서 최신 버전이 started 상태인지 확인
  • 홈페이지 로딩 후 CSS/JS 정상 반영 여부 확인
  • 프로젝트 카드 클릭 시 상세 페이지 라우팅 동작 확인
  • 운영 URL 직접 접속: https://base-project.fly.dev/
